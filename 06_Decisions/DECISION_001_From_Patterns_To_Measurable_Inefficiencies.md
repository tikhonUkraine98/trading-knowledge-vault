# DECISION_001 From Patterns To Measurable Inefficiencies

## Decision

Stop treating the project as a search for a beautiful chart pattern.

The project should search for measurable market properties that can pass statistical and execution validation.

## Old model

```text
Find a profitable pattern
```

## New model

```text
Market property
↓
Metric
↓
Statistical validation
↓
Execution validation
↓
Edge
```

## Reason

Pattern is only statistical repeatability. It is not proof of profitability.

Edge exists only after:

- fees;
- spread;
- slippage;
- execution losses.

## Consequence

Every candidate idea should answer:

- what exactly is measured?
- who pays?
- why does this repeat?
- how long does the opportunity live?
- can it survive execution?
