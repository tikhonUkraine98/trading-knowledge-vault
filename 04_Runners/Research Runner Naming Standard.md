# Research Runner Naming Standard

## Purpose

All research workflows, artifacts, and completion issues must be named clearly.

## Core format

DIRECTION_TASK_SCOPE_VERSION

Optional run suffix:

RUN001, RUN002, RUN003

## Direction prefixes

### SCALP

Scalping, DOM, tape, order-flow, microstructure signals inside one exchange.

### ARB

Cross-exchange arbitrage, spread charts, exchange lag, executable spread, inventory and rebalance research.

### MICRO

General market microstructure research.

### DISCOVERY

Wide search with unclear target.

### VALIDATION

Validation of previous findings.

### REPLAY

Post-factum execution validation.

## Practical rule

Before creating a workflow, choose:

- DIRECTION
- TASK
- SCOPE
- VERSION
- RUN_ID

Then use the same identifiers in workflow name, file name, run id, output folder, artifact name, issue title, and issue body.
