# DOM vs Tape

## Суть

[[Order Book]] / DOM показує наміри учасників: які заявки стоять у стакані.

[[Tape]] показує дії учасників: які угоди реально виконані.

Канонічна формула:

```text
Order Book = intentions
Tape = execution
```

## Практичний висновок

Велика заявка в стакані сама по собі не є сигналом. Потрібне підтвердження через виконані угоди.

Сильніший сигнал:

```text
Large bid
+
Aggressive buying in tape
```

Слабший сигнал:

```text
Large bid
+
No executions
```

## Значення для бота

Бот має збирати одночасно:

- стакан;
- стрічку угод;
- spread;
- derived metrics.

Після цього можна порівнювати, що виникає раніше:

- дисбаланс у стакані;
- агресивний потік у стрічці;
- рух ціни.
